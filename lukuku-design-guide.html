Voice & Tone
Copy Guidelines
DO
"68개 거래처. 97개 테스트케이스."
숫자가 말하게 하세요.
"브랜드는 브랜드에 집중하세요."
짧고 직접적으로.
"We ship. Every week."
영어 혼용 OK. 빌더 에너지.
DON'T
"최고의 파트너가 되어 드리겠습니다"
추상적인 약속 금지.
"귀사의 성공적인 해외 진출을 위해..."
기업 브로슈어 톤 금지.
Serif fonts (Playfair Display etc.)
산세리프 + 모노스페이스만.
Core rules: 숫자가 말하게 하세요 · 짧고 직접적으로 · 기술 용어를 두려워하지 마세요

Copywriting
카피 작성 원칙
한 문장에 하나의 메시지

숫자가 먼저, 설명이 뒤에

"MSA", "ERP I/F", "3PL" — 이 단어 자체가 역량 증명. 아는 사람에게 보여주세요.

Tokens
Token Architecture
Layer 1: Primitive
Raw values. Name = value.
--ink: #231F20
--mint: #2BF5A0
--white: #FFFFFF
--gray-100: #F5F5F5
--gray-500: #737373
--error: #EF4444
→
Layer 2: Semantic
Purpose-driven aliases.
--color-text-primary: var(--ink)
--color-bg-default: var(--white)
--color-bg-subtle: var(--gray-100)
--color-accent: var(--mint)
--color-border: var(--gray-200)
--color-error: var(--error)
→
Layer 3: Component
Scoped to components.
--btn-primary-bg: var(--color-text-primary)
--btn-accent-bg: var(--color-accent)
--card-default-border: var(--color-border)
--stat-highlight: var(--color-accent)
--status-active: var(--color-accent)
--code-bg: var(--color-bg-subtle)
ref: Toss TDS 3-layer + Vercel Geist token system
